# クイックスタートガイド

## 🎯 どの構成を選ぶべきか？

### **PoC Stack（AllInOne構成）を選ぶべきケース**

```bash
# bin/poc-app.ts を使用
cdk deploy -a "npx ts-node bin/poc-app.ts" --context env=dev
```

| 条件 | 説明 |
|------|------|
| 📦 **プロジェクト規模** | 小規模（< 50リソース） |
| 👥 **チーム人数** | 1-2人 |
| 📅 **プロジェクト期間** | 3ヶ月以内のPoC、MVP |
| 🚀 **デプロイ頻度** | 月1-2回 |
| 💰 **予算** | 少額（$10-50/月） |
| 🎯 **目的** | 素早くプロトタイプを作りたい |

**メリット**:
- ✅ シンプルで理解しやすい
- ✅ すぐに始められる
- ✅ 管理が容易

**詳細**: [POC_SETUP_GUIDE.md](./POC_SETUP_GUIDE.md)

---

### **8スタック構成を選ぶべきケース**

```bash
# bin/app.ts を使用（デフォルト）
cdk deploy --all --context env=dev
```

| 条件 | 説明 |
|------|------|
| 📦 **プロジェクト規模** | 中〜大規模（50+リソース） |
| 👥 **チーム人数** | 3人以上 |
| 📅 **プロジェクト期間** | 3ヶ月以上の長期運用 |
| 🚀 **デプロイ頻度** | 週1回以上 |
| 💰 **予算** | 中〜大規模（$100+/月） |
| 🎯 **目的** | 本番環境、スケーラブルなシステム |

**メリット**:
- ✅ 部分デプロイ可能（高速）
- ✅ チーム開発が容易
- ✅ 影響範囲の最小化
- ✅ データベースとS3を独立管理

**詳細**: [README.md](./README.md)

## 🚀 クイックスタート

### **Option 1: PoC Stack（推奨：初めての方）**

```bash
# 1. インストール
npm install

# 2. ビルド
npm run build

# 3. デプロイ
cdk deploy -a "npx ts-node bin/poc-app.ts" --context env=dev

# デプロイ時間: 約15-20分
# リソース数: 約40-50個
# 月額コスト: $50-100
```

### **Option 2: 8スタック構成（推奨：本番環境）**

```bash
# 1. インストール
npm install

# 2. ビルド
npm run build

# 3. デプロイ
cdk deploy --all --context env=dev

# デプロイ時間: 約25-40分
# リソース数: 約60個
# 月額コスト: $100-200
```

## 💡 FAQ

### **Q: どちらを選べば良いか迷っています**

**A**: 迷ったら **PoC Stack** から始めてください。

- シンプルで学習コストが低い
- 必要になったら8スタック構成に移行できる
- 移行ガイドが用意されている

### **Q: PoC Stackから8スタック構成に移行できますか？**

**A**: はい、可能です。

[MIGRATION_GUIDE.md](./MIGRATION_GUIDE.md) に詳細な手順があります。
ただし、データ移行が必要で、2-3時間のダウンタイムが発生します。

### **Q: 両方同時に使えますか？**

**A**: はい、可能です。

```bash
# PoC Stack（dev環境）
cdk deploy -a "npx ts-node bin/poc-app.ts" --context env=dev

# 7スタック構成（prod環境）
cdk deploy --all --context env=prod
```

異なる環境名を使えば、両方同時に運用できます。
